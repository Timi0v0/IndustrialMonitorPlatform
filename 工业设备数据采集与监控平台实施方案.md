# 工业设备数据采集与监控平台实施方案

> **项目定位：**基于 ASP.NET Core + WPF 构建一套工业设备数据采集、监控、报警、历史查询和接口开放平台。  
> **核心价值：**展示设备通信、后台采集服务、实时推送、WPF 客户端、数据库存储、报警处理、报表导出和系统架构能力。

---

## 1. 项目目标

本项目模拟真实工业现场中的设备数据采集与监控场景，实现从设备接入、数据采集、状态监控、异常报警、历史查询到第三方系统接口开放的完整链路。

最终效果：

- 启动设备模拟器后，系统可以模拟多台工业设备实时上报数据。
- ASP.NET Core 服务端负责采集、解析、存储、报警判断和接口开放。
- WPF 客户端可以实时查看设备状态、数据曲线、报警信息和历史数据。
- 支持通过 Web API 模拟 MES 或第三方系统对接。
- 支持报表导出，体现工业项目中的数据追溯能力。

## 2. 技术栈选型

### 后端技术

| 类型     | 技术                                |
| -------- | ----------------------------------- |
| 后端框架 | ASP.NET Core Web API                |
| 后台采集 | BackgroundService / Worker Service  |
| 实时通信 | SignalR                             |
| ORM      | Entity Framework Core               |
| 数据库   | SQLite，后续可扩展 MySQL / InfluxDB |
| 日志     | Serilog                             |

| 参数校验 | FluentValidation                    |

| 类型           | 技术                                |

| UI 控件库      | MaterialDesignInXamlToolkit         |
| 图表           | LiveCharts2                         |
| 实时通信客户端 | Microsoft.AspNetCore.SignalR.Client |
| API 调用       | HttpClient / Refit                  |

| 类型       | 技术                      |

## 4.2 数据采集模块

### 功能说明

服务端通过后台任务定时采集设备数据，支持 TCP Socket 第一版实现，后续扩展 Modbus TCP、MQTT。

### 采集流程

```
读取已启用设备配置
根据协议类型创建采集客户端
连接设备
读取数据
解析数据
保存实时数据
判断报警
通过 SignalR 推送到 WPF
```

### 采集数据模型

```

DeviceCode
CollectTime
Temperature
Voltage
Current
Pressure
RunStatus

RawData
```

## 第三阶段：设备模拟器

### 目标

实现一个 TCP 设备模拟器，用来模拟工业设备实时上报数据。

### 任务

- 创建 TCP Server 或 TCP Client 模拟器

- 支持配置设备编号

- 定时生成温度、电压、电流、压力数据

- 支持模拟正常数据

- 支持模拟异常数据

- 支持模拟设备断线

- 输出通信日志

### 模拟数据格式

```
{
  "deviceCode": "DEV001",
  "collectTime": "2026-06-14 10:00:00",
  "temperature": 35.6,
  "voltage": 12.4,
  "current": 5.2,
  "pressure": 0.6,
  "runStatus": "Running"
}
```

### 验收标准

- 模拟器可以持续发送数据
- 可以模拟多台设备

- 可以模拟设备断开连接
